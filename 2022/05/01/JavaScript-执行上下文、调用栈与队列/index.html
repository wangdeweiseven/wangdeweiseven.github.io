执行上下文

什么是执行上下文

执行上下文（Execution Context）是 JavaScript 代码被评估和执行时的环境概念。

执行上下文的类型

  1. 全局执行上下文

    • 默认的、基础的执行上下文
    • 创建全局对象（浏览器中为 window，Node.js 中为 global）
    • 将 this 指向全局对象

  2. 函数执行上下文

    • 每次函数调用时创建
    • 每个函数都有自己的执行上下文

  3. Eval 执行上下文

    • 在 eval 函数内部的代码有自己的执行上下文

执行上下文的生命周期

  1. 创建阶段

    • 创建变量对象（VO）
      • 函数参数（函数上下文）
      • 函数声明（优先）
      • 变量声明（初始值为 undefined）
    • 建立作用域链
    • 确定 this 指向

  2. 执行阶段

    • 变量赋值
    • 函数引用
    • 执行代码

  3. 回收阶段

    • 执行完毕后，等待垃圾回收

调用栈

什么是调用栈

调用栈（Call Stack）是一种数据结构，用于记录代码执行的位置。

调用栈的特点

  1. 后进先出（LIFO）结构
  2. 同步执行
  3. 栈溢出：当超过最大调用栈大小时会抛出错误

事件循环与任务队列

JavaScript 运行时模型

  1. 调用栈：同步代码执行
  2. Web APIs：浏览器提供的异步功能（setTimeout, DOM 事件等）
  3. 任务队列：回调函数等待执行
  4. 事件循环：监控调用栈和任务队列

任务队列类型

  1. 宏任务队列（Macrotask Queue）

    • setTimeout
    • setInterval
    • setImmediate (Node.js)
    • I/O 操作
    • UI 渲染
    • 事件回调

  2. 微任务队列（Microtask Queue）

    • Promise.then/catch/finally
    • MutationObserver
    • process.nextTick (Node.js)

事件循环流程

  1. 执行同步代码（调用栈）
  2. 遇到异步任务，交给 Web APIs 处理
  3. 同步代码执行完毕，调用栈为空
  4. 检查微任务队列，执行所有微任务
  5. 如有必要，进行 UI 渲染
  6. 从宏任务队列取出一个任务执行
  7. 重复上述过程
